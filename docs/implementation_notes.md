## Commit ad52a7ad8f5bc91029ea52eb9b5d5b5fb71f99ef

Design Flaw: There is an issue in the code base where a request for a button is captured and stored. The system keeps seeing the button press as unhandled, and sending the command repeatedly. The behavior needs to treat the  button press as a request, not an entity in the datastore, where the request has a unique ID and state related to how the request has been handled, i.e NEW, COMPLETED, REJECTED, PENDING, etc. The cooldown should be applied to the ability to ingest a new request, such that a user will get a rejected notification if they continuously trigger the button. The handling of the request should start within monitor, where the state is created.
